## 2026-05-30 - Add Zod Validation to Async POST Routes
**Vulnerability:** Mass assignment and unhandled promise rejections on validation failures (potential DoS). Unvalidated `req.body` data was being passed directly to `storage.createX()` functions in several async route handlers (e.g., `/api/support/tickets`, `/api/referrals`, `/api/projects/:projectId/dprs`). Using Zod's `.parse()` without a `try/catch` block in async handlers causes unhandled promise rejections that crash the Node.js process if invalid data is sent.
**Learning:** It is crucial to not only validate input but also to safely handle synchronous validation errors within async Express handlers to prevent denial-of-service vulnerabilities.
**Prevention:** Always wrap Zod `.parse()` calls in a `try/catch` block (or use `.safeParse()`) in async route handlers to return a structured error response (`400 Bad Request`) instead of crashing the server.
